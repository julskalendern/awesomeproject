#Правила наименования веток
Помимо main есть 3 основные ветки:
develop - для разработки
test - для тестирования
hotfix - для сроных исправлений
Правила наименования:
Для develop - название функционала, например,
develop/authotisation для авторизации
develop/money для денежных операций ит д.
Для test -название + дата тестирования
test/authorization/2026-02-02
Для hotfix - дата срочного исправления

#как устроен процесс внесения своих изменений:
Изменение должно пройти такие шаги: develop -> test -> main
Если тестирование неудачное, изменения несрочные, то порядок такой: develop 
-> test -> develop
Если тестирование неудачное, изменения срочные, то develop -> test -> hotfix

!!!
Перед внесением изменений "главный по больнице" должен убедиться,
что ветка для фичи есть. Иначе он должен ее создать с помощью git branch
!!!

Алгоритм внесения изменений:
0)клонируем репозиторий с помощью git clone
1)переходим на ветку, в которой будут вноситься изменения, например
git checkout  develop/[название фичи]
2) после внесения необходимых изменений выполняем
git add .
git commit -m "[название фичи] описание изменений"
3) добавляем на удаленный репозиторий c помощью
git push origin develop/[название фичи]  
4) В github создаем pullrequest в test с указанием фичи и  описанием изменений
5)Если все ок, то тестирующие сотрудники мерджат с main

#Инструкция для неофитов:
Для каждого новичка "старший по подъезду" создает свою ветку вида
develop/[название фичи]/[фамилия неофита], например
develop/money/ivanov

 0) клонируем репозиторий git clone https://github.com/julskalendern/awesomeproject
 1) переходим в созданную для вас ветку с помощью git checkout [название скажет взрослый]
 2) вносим необходимые изменения
git add .
git commit -m "[название фичи]-[тег взрослого]-описание изменения"
 3) грузим на удаленный репозиторий
git push origin develop/[название фичи]/[твоя фамилия]
 4) на GitHub создаем pullrequest с названием ветки и описанием.
 5) если взрослый не нашел к чему придраться, то делаем слияние со взрослой же веткой
git merge develop/[название фичи]
 6) пушим
git push origin develop/[название фичи]
